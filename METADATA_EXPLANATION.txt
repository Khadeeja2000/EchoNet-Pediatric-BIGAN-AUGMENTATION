WHAT IS METADATA? - Simple Explanation
================================================================================

METADATA = Information ABOUT something, not the thing itself

REAL-WORLD EXAMPLES:
================================================================================

1. MUSIC FILE:
   - The actual music = The audio data (the song itself)
   - Metadata = Title, Artist, Album, Year, Genre
   - Metadata is stored separately (like in iTunes)

2. PHOTO:
   - The actual photo = The image pixels
   - Metadata = Date taken, Camera model, Location, File size
   - Metadata is stored in the file (EXIF data)

3. BOOK:
   - The actual book = The story/content
   - Metadata = Title, Author, ISBN, Publication date, Pages
   - Metadata is on the cover/back

YOUR PROJECT:
================================================================================

VIDEO FILE:
- The actual video = The moving pictures (frames, pixels)
- This is what you SEE when you play the video

METADATA (stored in CSV file):
- View type (A4C or PSAX)
- Patient sex (Male or Female)
- Age bin (0-1, 2-5, etc.)
- BMI category
- Ejection fraction
- File path
- Other patient information

WHERE IS METADATA STORED?
================================================================================

In your project, metadata is stored in:
  data/processed/manifest.csv

This CSV file has columns like:
  - file_path (where video is stored)
  - view (A4C or PSAX)
  - sex (M or F)
  - age_bin (0-1, 2-5, etc.)
  - bmi (BMI category)
  - ef (ejection fraction)
  - etc.

HOW THE SYSTEM USES IT:
================================================================================

1. Video file = The actual echocardiogram video (moving pictures)
2. CSV file = Contains metadata ABOUT each video
3. System reads metadata from CSV (not from video pixels)
4. Formats metadata as description
5. Uses it for validation

EXAMPLE:
================================================================================

Video file: CR32a95e8-CR32a9abd-000032.mp4
  ↓ (This is the actual video - moving pictures)

CSV metadata for this video:
  - view: PSAX
  - sex: F
  - age_bin: 0-1
  - ef: 43.04
  ↓ (System reads this from CSV)

Generated description:
  "Echocardiogram video: PSAX view, Female patient, age 0-1 years"
  ↓ (Created from metadata, not video analysis)

WHY THIS MATTERS:
================================================================================

The system is NOT analyzing the video pixels to determine:
- Patient sex
- Age
- View type

Instead, it's READING this information from the CSV file (metadata).

This is why it "guesses correctly" - it's reading from the same source!

WHAT WOULD BE REAL VIDEO ANALYSIS:
================================================================================

To actually analyze video content, you would need:
- Look at video pixels/frames
- Extract features from images
- Use computer vision/deep learning
- Train a model to recognize patterns
- Determine demographics from visual features

This would be much harder and require:
- Trained models
- Large datasets
- Complex algorithms
